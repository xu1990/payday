
# 薪日 PayDay - 产品需求文档 v1.1（优化版）

> 本版本基于 v1.0 进行结构、业务闭环、合规、安全与可落地性优化，重点增强：用户路径闭环、商业化预留、风控与隐私合规、数据指标体系与技术扩展性。

---

## 一、产品概述（优化）

### 1.1 产品定位
**薪日 PayDay** —— 面向职场人群的「发薪情绪 + 轻财务记录 + 匿名社区」综合平台，核心是围绕“发薪日”这一强情绪节点构建工具 + 社交双轮驱动产品。

### 1.2 目标用户细分
| 用户类型 | 特征 | 核心诉求 |
|--------|------|---------|
| 初入职场 | 工资低、焦虑 | 情绪表达、共鸣 |
| 稳定打工人 | 收入稳定 | 记录、统计、理财意识 |
| 自由职业 | 收入波动 | 多收入管理 |
| 兼职人群 | 多发薪日 | 多配置能力 |

### 1.3 核心价值优化
- **情绪价值**：发薪日仪式感、打卡、海报分享
- **工具价值**：工资记录 → 趋势分析 → 预算意识
- **社交价值**：匿名安全表达
- **数据价值（新增）**：行业/城市工资洞察

### 1.4 商业化预留（新增）
- 虚拟勋章 / 主题皮肤
- 发薪日专属海报模板
- 高级数据分析（月度报告）
- 品牌合作（非强广告）

## 二、整体产品结构优化

### 2.1 用户核心路径（新增）

```
首次进入
 → 设置发薪日
 → 记录第一次工资
 → 查看统计
 → 进入社区共鸣
 → 分享海报
 → 形成留存
```

### 2.2 功能分层（优化）
| 层级 | 目标 | 模块 |
|------|------|------|
| 工具层 | 留存 | 发薪日、工资记录、统计 |
| 情绪层 | 活跃 | 心情、打卡、海报 |
| 社交层 | 增长 | 社区、评论、关注 |
| 数据层 | 价值 | 行业统计、趋势 |
| 商业层 | 变现 | 会员/主题 |

## 三、小程序端优化点

### 3.1 首页优化建议
新增：
- 发薪日打卡按钮
- 月度收入对比上月
- 今日社区热帖推荐 1 条
- “距离下次发薪还能买几杯咖啡”趣味文案

### 3.2 工资记录模块优化

#### 新增字段建议
| 字段 | 目的 |
|------|------|
| 税前金额 | 统计更真实 |
| 扣税金额 | 用户认知 |
| 收入来源标签 | 多职业支持 |
| 是否延迟发薪 | 形成统计 |

#### 统计优化
- 月均收入
- 收入波动率
- 发薪准点率排行

## 四、社区模块优化

### 4.1 风险与合规强化（重要）
新增合规点：
- 未成年人限制
- 工资截图自动打码（手机号/身份证）
- 地理位置模糊化（城市级）
- 用户一键清空数据

### 4.2 内容生态优化
新增机制：
- 热帖冷却机制（防刷）
- 标签权重排序
- 新人保护流量
